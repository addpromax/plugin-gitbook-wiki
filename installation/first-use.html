第一次使用

第一次使用

恭喜您成功安装并配置了 Codex 插件！本章将指导您体验插件的核心功能，帮助您快速上手。

开始体验

1. 打开主界面

在游戏中输入命令打开 Codex 主界面：

/codex

您将看到一个包含以下内容的GUI界面：

  • 📚 各种图鉴分类（物品收集、怪物猎人等）
  • 🎯 每个分类的解锁进度
  • ⭐ 可用的发现条目

2. 体验第一个发现

物品发现体验

  1. 获取钻石

    /give @s diamond 1

  2. 观察反馈

    • 🎵 听到提示音效
    • 💬 收到发现消息："&e发现新物品：&b第一颗钻石"
    • ⭐ 获得经验奖励

  3. 查看图鉴更新

    • 重新打开 /codex 界面
    • 点击 "物品收集" 分类
    • 查看新解锁的钻石发现条目

怪物击杀体验

  1. 生成测试怪物

    /summon zombie ~ ~ ~

  2. 击杀怪物

    • 使用武器击杀僵尸
    • 观察击杀反馈和奖励

  3. 查看更新

    • 打开图鉴的 "怪物猎人" 分类
    • 查看新的击杀记录

3. 探索自动图鉴

如果您启用了自动图鉴功能：

附魔图鉴

  1. 获取附魔物品

    /give @s enchanted_book{StoredEnchantments:[{id:"minecraft:sharpness",lvl:1}]}

  2. 查看附魔图鉴

    • 在主界面点击 "附魔图鉴"
    • 浏览已发现的附魔
    • 查看详细的附魔信息

钓鱼图鉴（需要 CustomFishing）

  1. 钓鱼体验

    • 使用钓鱼竿进行钓鱼
    • 每钓到新鱼类都会自动记录

  2. 查看钓鱼图鉴

    • 在主界面点击 "钓鱼图鉴"
    • 查看鱼类统计和最大尺寸记录

分类界面功能

点击任意分类后，您可以：

  1. 浏览发现条目

    • 已解锁：显示详细信息和获得时间
    • 未解锁：显示模糊提示

  2. 使用分页功能

    • ⬅️ 上一页按钮
    • ➡️ 下一页按钮
    • 📄 页数指示器

  3. 查看详细信息

    • 将鼠标悬停在物品上
    • 查看完整的描述和统计信息

常用命令

基础命令

命令 功能 权限
/codex 打开主界面 codex.command.main
/codex reload 重载配置 codex.admin
/codex help 显示帮助 codex.command.main

管理员命令

命令 功能 权限

/codex database info 数据库信息 codex.admin
/codex player <玩家> 查看玩家数据 codex.admin

自定义体验

调整奖励

编辑分类配置文件中的奖励设置：

rewards:
  per_discovery:
    - "playsound: ENTITY_PLAYER_LEVELUP;1;1"          # 音效
    - "console_command: experience add %player% 50 points"  # 经验
    - "message: &e恭喜！解锁新发现：&f%name%"            # 消息
    - "title: 10;40;10;&6发现！;&e%name%"              # 标题

添加更多发现

在分类配置文件中添加新的发现条目：

discoveries:
  新发现ID:
    name: "&6新发现名称"
    description:
      - "&7这是一个新的发现！"
    discovered_on:
      type: ITEM_OBTAIN  # 或其他触发类型
      value:
        item_type: EMERALD  # 触发条件

性能监控

检查插件状态

使用以下命令监控插件性能：

/codex debug

这会显示：

  • 📊 内存使用情况
  • 📈 数据库状态
  • 🔗 插件挂钩状态
  • ⏱️ 响应时间统计

优化建议

  1. 数据存储

    • 小型服务器：使用 YAML 文件
    • 中型服务器：使用 H2 数据库
    • 大型服务器：使用 MySQL 数据库

  2. 配置调优

    # 调整自动保存频率
auto_save_interval: 15  # 分钟

# 启用内存清理
memory_cleanup: true
cleanup_interval: 1800  # 秒

故障排除

常见问题快速修复

问题：发现没有被触发

检查步骤：

  1. 确认配置文件语法正确
  2. 检查触发条件是否匹配
  3. 查看控制台是否有错误信息

问题：GUI显示异常

检查步骤：

  1. 确认 inventory.yml 配置正确
  2. 检查物品ID是否有效
  3. 验证颜色代码格式

问题：奖励不生效

检查步骤：

  1. 验证权限配置
  2. 检查命令格式
  3. 确认PlaceholderAPI变量
